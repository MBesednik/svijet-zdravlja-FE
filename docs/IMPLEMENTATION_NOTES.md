# Implementacijske napomene

Blog MVP koristi 'localStorage' (ključ 'svz_blog_posts') i data-URL vrijednosti kako bi se objave i pripadne naslovne slike čuvale u potpunosti na klijentu bez servera. To je pogodno za prototip jer omogućuje instant CRUD iskustvo bez backenda, ali povećava potrošnju memorije (posebno zbog kodiranih slika) te je nepouzdano za veće timove. Za prijelaz na API sloj potrebno je helper funkcije u 'scripts/blog.js' ('getPosts', 'savePosts', 'createPost', 'updatePost', 'deletePost') zamijeniti HTTP pozivima prema backendu. Funkcija 'fileToDataURL' tada služi samo za preliminarnu validaciju prije slanja originalne slike prema API-u; backend treba vratiti URL (npr. 'images/blog/...') koji se upisuje u polje 'featuredImage' umjesto data-URL vrijednosti, čime se zadržava ista struktura objekta u sučelju.
